GitHub

Веб-сервис основан на системе контроля версий Git и разработан на Ruby on Rails и Erlang компанией GitHub, Inc (ранее Logical Awesome). Сервис бесплатен для проектов с открытым исходным кодом и (с 2019 года) небольших частных проектов, предоставляя им все возможности (включая SSL), а для крупных корпоративных проектов предлагаются различные платные тарифные планы.

Слоган сервиса — «Social Coding» — на русский можно перевести как «Пишем код вместе». На футболках же печатают совсем другую фразу: «Fork you!» («Ответвись!»). С одной стороны, она созвучна с англоязычным ругательством и намекает на неформальную атмосферу. С другой, эти слова напоминают, что создавать новые форки с Git можно легко и безболезненно — традиционно, к созданию веток разработчики проектов с открытым исходным кодом относятся негативно — а также созвучна названию одной из возможностей GitHub — очереди форков.

git init

Kоманда, с которой можно начать работу с Git.Чтобы создать git репозиторий, выполняется команда git init.

git status

Команда git status используется, чтобы проверить текущее состояние репозитория, а также, для продолжения отслеживания состояния репозитория и рабочего каталога.

git add

Команда используется, чтобы добавить изменения в репозиторий. Если было одно изменение, можно написать git add и название файла, который был изменен. Если было несколько изменений - делае запись git add . - и все изменения сохраняются.

git checkout

Команда используется для переключения веток и выгрузки их содержимого в рабочую директорию. Также используется для отмены текущих изменений в проекте.

git commit

Команда git commit берёт все данные, добавленные в индекс с помощью git add, и сохраняет их слепок во внутренней базе данных, а затем сдвигает указатель текущей ветки на этот слепок

git push

Команда git push используется для установления связи с удалённым репозиторием, вычисления локальных изменений отсутствующих в нём, и собственно их передачи в вышеупомянутый репозиторий. Этой команде нужно право на запись в репозиторий, поэтому она использует аутентификацию.

git remote

Команда git remote служит для управления списком удалённых репозиториев. Она позволяет сохранять длинные URL репозиториев в виде понятных коротких строк, например «origin», так что вам не придётся забивать голову всякой ерундой и набирать её каждый раз для связи с сервером. Вы можете использовать несколько удалённых репозиториев для работы и git remote поможет добавлять, изменять и удалять их.

git branch

Команда git branch — это своего рода "менеджер веток". Она умеет перечислять ваши ветки, создавать новые, удалять и переименовывать их.

pwd

Отображает текущее местоположение в структуре каталогов.

ls

Выводит информации о том, что находится в папке.

cd

Позволяет перейти в новый каталог. cd и пишем путь к папке

cat

Позволяет посмотреть содержимое файла

mkdir

Создаёт новый каталог в текущем каталоге.

mv

Перемещение файла в папку.

man

Отображает руководства по командам.

nano

Простой текстовый редактор.Позволяет создать и изменить фаил.

touch

Эта команда создает текстовый фаил.

cp

Копирование файла.
